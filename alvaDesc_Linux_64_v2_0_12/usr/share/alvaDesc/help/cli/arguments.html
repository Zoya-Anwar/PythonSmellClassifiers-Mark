3.1 Arguments

Three alternative arguments can be used to run alvaDescCLI:

Argument Description
--script=<SCRIPT FILE> or -s <SCRIPT FILE> All the available options are contained in the SCRIPT FILE. When this is used no other argument can be present on the command-line except for --verbose and --threads
--iSMILES=<SMILES> Uses a single SMILES string as input
--input[=<INPUT FILE>] It can be used to input a set of molecules either with an INPUT FILE or in the standard input (default)

Calculation arguments:

Argument Description
--inputtype=SMILES|MDL|SYBYL|HYPERCHEM Input file type. It’s mandatory if the input file name is omitted
--descriptors=<DESCRIPTOR NAMES> List of comma-separated descriptors to be calculated. To calculate all use –descriptors=ALL. To calculate all 2D use –descriptors=ALL2D.
--labels Adds descriptor and molecule labels
--ecfp Calculates extended connectivity fingerprint
--pfp Calculates path fingerprint
--size=<SIZE> Size of hashed fingerprint (default: 1024)
--min=<MIN LENGTH> Minimum fragment length for hashed fingerprint (default: 0)
--max=<MAX LENGTH> Maximum fragment length for hashed fingerprint (default: 2)
--count=<TRUE|FALSE> Counts fragments for hashed fingerprint (default: TRUE)
--bits=<BITS PER PATTERN> Bits per pattern for hashed fingerprint (default: 2)
--fpoptions=<OPTIONS> Atom types for hashed fingerprint (default: Atom type, Aromaticity, Charge, Connectivity (total), Bond order)
--maccsfp Calculates MACCS166 fingerprint
--output[=<OUTPUT FILE>] Output file (default: standard output)
--threads=<NUMBER OF THREADS> Number of threads used in the calculation (default: equal to the maximum available CPUs)

Other arguments:

Argument Description
--descriptorlist Prints the list of available descriptors
--cite Prints the citation for alvaDesc
--help or -h Prints the alvaDescCLI help
--verbose Prints verbose messages during execution
--update[=<UPDATE FILE>] Checks if a software update is available. If a file name is passed tries to download it.
--license[=<LICENSE FILE>] Shows information of the given license file (default: the active one)
--request-license=<REQUEST FILE> Generates license file request
--import-license=<LICENSE FILE> Imports a license file
